# Security Policy

## Supported Versions

Security and correctness fixes are applied to the latest published release. The immutable `v1.0.0` tag and its assets are not silently replaced; a correction is published under a new semantic version.

## Reporting

Report suspected vulnerabilities privately through the repository's GitHub Security Advisory interface. Do not include credentials, proprietary market data, or personally identifiable information in a public issue.

Include the affected commit or release, platform, reproduction steps, expected and observed behavior, and potential impact. Reports concerning financial accounting, provenance bypass, archive traversal, data-boundary leakage, dependency integrity, or deterministic reconstruction are treated as security-relevant.

The project is a research system, not a live-trading service. No operational trading credentials should be configured or stored in this repository.
